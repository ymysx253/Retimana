# アプリの詳細
- **名前**: Retimana（レティマナ）
- **概要**: 観光客向けリアルタイムバス案内PWA（Webアプリ）。
  - 施設名・観光スポット名で行き先を検索
  - 現在地から最寄りバス停Top3を自動抽出
  - 目的地行きのバスをリアルタイムで地図に表示
  - 「どれに乗ればいいか」を一目で表示（乗り間違い防止）
  - 投げ銭（任意支援）機能でインバウンド向け収益化
- **対象端末**: iPhoneスマホ、Androidスマホ（PWA）

# 環境
- **ホスティング**: さくらインターネット
- **開発環境**: ASP.NET Core（.NET 9）
- **言語**: C#
- **開発サポート**: Claude Code
- **ソース管理**: GitHub

# ブランチ命名規則
- フォーマット: `<種別>/<YYYYMMDD>`
- 種別: `feature/`（新機能）、`fix/`（バグ修正）、`refactor/`（内部改善）、`docs/`（ドキュメント）
- 日付は着手日基準（翌日に持ち越してもブランチ名は変更しない）
- 同日に複数作業する場合は末尾に `-1`, `-2` を付ける
- 例: `feature/20260505`、`feature/20260505-2`
- 作業内容の詳細はコミットメッセージで残す
- マージ済みのブランチはローカル・リモート両方削除する

# 作業フロー（Claude Code 向け）
**作業開始の指示があったら**（例: 「作業を始めます」「実装を始めて」など）:
1. 種別（feature/fix/refactor/docs）を確認
2. `git checkout -b <種別>/<YYYYMMDD>` でブランチを切る
3. 以降の作業はそのブランチで進める

**作業終了の指示があったら**（例: 「今日はここまで」「終わり」「コミットして」など）:
1. 変更を `git add` → `git commit`（コミットメッセージは下記規則に従う）
2. `main` にチェックアウト → `git merge <作業ブランチ>`
3. `git push origin main`
4. `git branch -d <作業ブランチ>` でローカル削除
5. `git push origin --delete <作業ブランチ>` でリモート削除

**作業途中**: コミット可否を逐一確認しない。明示的に終了指示が出るまで作業ブランチで作業を続ける。

# コミットメッセージ
- 1行目: 要約（50文字以内、日本語OK）
- 必要に応じて空行を挟んで詳細を記述
- 「何を」だけでなく「なぜ」も書く
- 例:
  ```
  バス停逆引きロジックに距離上限を追加

  500m超のバス停を含めると徒歩時間が現実的でなくなるため、
  Haversine計算後に500m以内のみ抽出するよう変更した。
  ```

# 設計書
- 設計書は [docs/](docs/) 配下に整理されている。索引は [docs/README.md](docs/README.md)
- フォルダ分類:
  - `docs/features/` — 画面・機能の設計書（home / area-select / i18n / gtfs-ingestion 等）
  - `docs/architecture/` — 横断的な技術設計（マップ全体設計、ODPT API 利用ガイド等）
  - `docs/design/` — デザイン素材（画像・ロゴ）
  - `docs/odpt/` — ODPT 外部資料（API仕様コピー・GTFS-JP サンプル）
  - `docs/retimana_concept.md` — トップ: ブランドコンセプト
- 新規追加・既存変更時はカテゴリの合うフォルダに置き、`docs/README.md` の索引にも追記する
- **新しい機能を作るとき、または既存機能を変更するときは必ず設計書を更新する**
- 設計書には以下を含める:
  - 概要
  - 画面・UI（対応するファイル）
  - ソース構成（ファイル一覧と役割）
  - 主要な関数・メソッド（引数・戻り値・役割）
  - データモデル
  - 外部依存
  - 更新履歴
