# 多言語対応（ローカライゼーション）

## 概要
アプリ全体の表示テキストを4言語（日本語・英語・中国語・韓国語）で切り替えられるようにする仕組み。
アプリ名「Retimana」だけは全言語固定。それ以外の文言はすべてリソースファイル経由で取得する。

採用方式: **ASP.NET Core 標準ローカライゼーション**（`IStringLocalizer<T>` + `.resx`）+ **クッキー保持**。

## サポート言語
| カルチャ | 言語 | 自言語名（プルダウンに表示） |
|----------|------|---------|
| `ja` | 日本語（デフォルト） | 日本語 |
| `en` | English | English |
| `zh` | 中文 | 中文 |
| `ko` | 한국어 | 한국어 |

## ファイル構成
```
Retimana/Resources/
├── SharedResources.cs          # 型マーカー（空クラス、namespace Retimana）
├── SharedResources.ja.resx     # 日本語文言
├── SharedResources.en.resx     # 英語文言
├── SharedResources.zh.resx     # 中国語文言
└── SharedResources.ko.resx     # 韓国語文言
```

全文言を1つの `SharedResources` にまとめる方針。キー名で意味を分類する（例: `Home.Start`, `AreaSelect.Title`）。

## キー一覧（現状）
| キー | ja | en | zh | ko |
|------|----|----|----|----|
| `Home.Start` | スタート | Start | 开始 | 시작 |
| `Home.LanguageAriaLabel` | 言語切替 | Language | 语言 | 언어 |
| `AreaSelect.Title` | 地方を選ぶ | Choose a region | 选择地方 | 지방 선택 |
| `AreaSelect.MapAlt` | 日本地図 | Map of Japan | 日本地图 | 일본 지도 |

新しい文言は4ファイルすべてに同じキー名で追加する。1つでも欠けると、その言語ではキー名がそのまま表示されてしまう。

## 配線

### 1. `Program.cs`
- `AddLocalization()` を呼ぶ（**`ResourcesPath` は指定しない**）
- `Configure<RequestLocalizationOptions>` で `SetDefaultCulture("ja")` + サポート4言語を登録
- `app.UseRequestLocalization()` でリクエストごとにカルチャを確定（クッキー → Accept-Language → デフォルト の優先順）
- `MapGet("/set-language", ...)` で言語切替エンドポイントを公開

#### なぜ ResourcesPath を設定しないか（重要）
.NET SDK の既定では、`Retimana/Resources/SharedResources.ja.resx` を埋め込むと、サテライトアセンブリ内のマニフェスト名は `Retimana.SharedResources.ja.resources`（"Resources" フォルダ名を含まない）になる。
`ResourcesPath = "Resources"` を設定すると `IStringLocalizer` は `Retimana.Resources.SharedResources.ja.resources` を探しに行き、見つからない → 全言語でキー名（`Home.Start` 等）がそのまま返される、という症状になる。
そのため `AddLocalization()` を引数なしで呼び、ルックアップ対象を `Retimana.SharedResources.<culture>.resources` に合わせる。.resx ファイル自体は `Resources/` フォルダに置いたままで問題ない（物理パスとマニフェスト名は無関係）。

### 2. カルチャ確定の優先順位（`UseRequestLocalization` 既定）
1. `.AspNetCore.Culture` クッキー（前回選択した言語）
2. `Accept-Language` ヘッダー（ブラウザ言語）
3. `SetDefaultCulture("ja")`

### 3. コンポーネントでの使い方
```razor
@inject IStringLocalizer<SharedResources> L

<a class="start-button" href="area">@L["Home.Start"]</a>
```

### 4. `<html lang>` 属性
`Components/App.razor` で `<html lang="@CultureInfo.CurrentUICulture.TwoLetterISOLanguageName">` と動的設定。
スクリーンリーダーや検索エンジンに現在のページ言語を正しく伝える。

### 5. 言語切替プルダウン（Home画面）
- `<select onchange="location.href='/set-language?culture='+...+'&returnUrl='+...">` でブラウザを直接ナビゲーション
- `<form>` を使わない理由: Blazor enhanced navigation がフォーム送信を横取りしうるため、`location.href` 代入で純粋なブラウザ全画面遷移を強制する
- サーバーはクッキーをセットして `LocalRedirect(returnUrl)` で戻す
- 結果: ページがリロードされ、`UseRequestLocalization` が新しいクッキーを読み、新言語で再描画される

## 動作シーケンス
1. 初回アクセス: クッキーなし → `Accept-Language` 参照 → 該当言語またはデフォルト `ja` で表示
2. ユーザーがプルダウンで言語を変更
3. `onchange="this.form.submit()"` で `/set-language?culture=en&returnUrl=/` に GET
4. サーバーが `.AspNetCore.Culture` クッキー（1年有効）をセット
5. `Results.LocalRedirect(returnUrl)` で元のページにリダイレクト
6. リクエスト時に `UseRequestLocalization` がクッキーを読んで新言語のカルチャを設定
7. ページが新言語で再描画される

## アプリ名「Retimana」の扱い
- 全言語で固定
- リソースファイルにキーは作らず、Razor 内に文字列リテラルで直接書く（`<PageTitle>Retimana</PageTitle>` 等）
- ロゴ画像 `retimana_logo.svg` も全言語で同一

## 言語名ラベル
プルダウンに表示する選択肢の言語名は **その言語で自分自身を表記**（慣習）:
- 日本語 / English / 中文 / 한국어
- リソースファイルに入れず、`Home.razor` のコードに直接配列で持つ（翻訳不要のため）

## 今後の拡張
- **URLプレフィックス対応**: `/ja/...` `/en/...` のようなURL形式を追加可能。クッキー方式と共存できる
- **言語自動検出のUX改善**: 初回アクセス時に検出言語と異なる場合の通知バー等
- **ページタイトルの言語付き**: 現状はキーごとに別キーで書いているが、テンプレ的に組み立ても検討
- **翻訳プラットフォーム連携**: `.resx` は Crowdin / Lokalise / POEditor 等で扱える
- **複数形・パラメータ**: `IStringLocalizer` は `L["Key", arg]` 形式で `{0}` 置換も対応可
