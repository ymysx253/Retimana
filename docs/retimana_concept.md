# Retimana（レティマナ）
## ブランドコンセプト & 名前の由来

---

## 名前の由来

**Retimana** はラテン語を語源とする4つのコンセプトワードを融合した造語です。

```
Re  →  Realis    ラテン語「本物・リアル」
Ti  →  Tempus    ラテン語「時間・タイミング」
ma  →  Mappa     ラテン語「地図」
na  →  Navigare  ラテン語「向かう・進む」
```

> **「本物のリアルタイム地図で、目的地へ進もう」**

英語圏には Real / Time / Map / Navigate として自然に伝わり、スペイン語・イタリア語圏にも語感が馴染み、アジア圏には音として覚えやすい、グローバルな名前です。

## ブランドコンセプト

### ミッション
観光客や旅行者が、バスや公共交通機関を**迷わず・楽しく・気軽に**使えるようにする。

### 3つの価値

| 価値 | 内容 |
|------|------|
| **便利** | 行き先を選ぶだけ。バス停名を知らなくても使える |
| **成長** | 利用者が増えるほど、より良いサービスへと進化する |
| **貢献** | 公共交通のオープンデータを活用し、地域の観光・移動を支援する |

### ターゲットユーザー
- 京都・日本を訪れる**インバウンド観光客**
- バス路線に不慣れな**国内観光客**
- 初めての路線を使う**すべての人**

## サービスの特徴

### 既存サービスとの違い

| 既存アプリ | Retimana |
|-----------|---------|
| バス停名・路線番号を知っている前提 | **行き先（施設名・観光スポット）を選ぶだけ** |
| 経路検索が主役 | **リアルタイムのバス位置が主役** |
| 情報量が多く複雑 | **「どれに乗ればいいか」だけを表示** |
| 乗り換え案内 | **「今ここから乗れるバス」に特化** |

### コア機能
1. **施設名・観光スポットで行き先を検索**
   - バス停名を知らなくても使える
   - OpenStreetMapと連携した施設データベース

2. **最寄りバス停Top3を自動表示**
   - 現在地から徒歩時間付きで表示
   - 乗り場番号をわかりやすく表示

3. **リアルタイムバス位置表示**
   - GTFS-RTを活用したリアルタイム接近情報
   - 「あと何分で来るか」が一目でわかる

4. **「これに乗ればOK」を明確に**
   - 最速で乗れる便を強調表示
   - 逆方向のバスは表示しない（乗り間違い防止）

5. **投げ銭（支援）機能**
   - Stripeによる任意支援
   - インバウンド向け多通貨・多言語対応
   - Alipay・WeChat Pay・Apple Pay対応

## 技術スタック

| 分野 | 技術 |
|------|------|
| バックエンド | ASP.NET Core（C#） |
| リアルタイムデータ | GTFS-RT（Protocol Buffers） |
| 静的データ | GTFS-JP（国土交通省標準フォーマット） |
| データ提供元 | 公共交通オープンデータセンター（ODPT） |
| 地図 | Leaflet + OpenStreetMap |
| 施設検索 | OpenStreetMap Nominatim API |
| リアルタイム配信 | SignalR（WebSocket） |
| 決済 | Stripe Payment Links |
| ホスティング | さくらインターネット |
| ドメイン | retimana.com |
